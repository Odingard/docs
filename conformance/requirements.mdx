# Conformance Requirements

To claim VERDICT WEIGHT™ conformance, a system must satisfy the following requirements. Requirements are categorized as MUST (baseline) or SHOULD (extended).

## Conformance Levels

**VERDICT WEIGHT™ Core** — Satisfies R1–R5. Baseline confidence synthesis guarantees.

**VERDICT WEIGHT™ Extended** — Satisfies R1–R9. Full adversarial robustness and operational maturity.

---

## Requirements Table

| ID | Level | Requirement |
|---|---|---|
| R1 | MUST | **Four-stream input** — System must ingest SR, CC, TD, and HA as independent inputs |
| R2 | MUST | **Geometric mean synthesis** — SS must be computed via context-weighted geometric mean of all four streams |
| R3 | MUST | **Doubt Index** — DI must be computed as normalized coefficient of variation across streams |
| R4 | MUST | **Consequence Weight** — CW must integrate SS and DI via the specified penalization formula |
| R5 | MUST | **Five-tier output** — System must produce CRITICAL/HIGH/MEDIUM/LOW/NOISE classifications |
| R6 | SHOULD | **Context profiles** — System should support at least one named domain profile with validated weights |
| R7 | SHOULD | **Temporal decay** — TD should be computed via exponential decay with domain-specific λ |
| R8 | SHOULD | **DI warning threshold** — System should surface explicit WARNING when DI > 0.70 |
| R9 | SHOULD | **Adaptive learning** — System should record outcome feedback for calibration improvement |

---

## R1: Four-Stream Input

A conformant system must accept SR, CC, TD, and HA as independent scalar inputs in [0,1]. These must be computed from source data — they cannot be the same underlying score aliased to four variables.

```python
# CONFORMANT
result = vw.score(SR=0.85, CC=0.12, TD=0.90, HA=0.78)

# NON-CONFORMANT (same score repeated)
result = vw.score(SR=0.85, CC=0.85, TD=0.85, HA=0.85)
```

---

## R2: Geometric Mean Synthesis

SS must be computed as the weighted geometric mean. Arithmetic mean or other aggregation methods do not satisfy R2.

$$SS = \prod_{i} s_i^{W_i}$$

---

## R3: Doubt Index

DI must reflect inter-stream disagreement. A system that sets DI=0 always, or that does not compute DI from stream variance, does not satisfy R3.

---

## R4: Consequence Weight

CW must decrease as DI increases. A system where CW = SS always does not satisfy R4.

---

## R5: Five-Tier Output

The output tier must map CW to one of exactly five named tiers with the specified thresholds. Custom threshold tuning is permitted within ±0.05 of the specification defaults.

---

## Claiming Conformance

Organizations implementing VERDICT WEIGHT™ in production systems may claim conformance after:

1. Running the [conformance test suite](./testing) against their implementation
2. Verifying all MUST requirements pass
3. Disclosing which SHOULD requirements are satisfied

Reference the conformance claim as:
```
Implements VERDICT WEIGHT™ Core [or Extended] Conformance
Per specification v3.0 — https://verdictweight.dev
```
