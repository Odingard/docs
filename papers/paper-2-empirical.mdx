## Abstract

Confidence-scoring frameworks are commonly proposed and rarely subjected to peer-review-grade empirical validation. This paper presents that validation for VERDICT WEIGHT, an eight-stream composition framework introduced in Paper 1. The empirical evaluation is structured around the seven attack categories most commonly applied by IEEE-grade reviewers: (1) head-to-head comparison against established baselines, (2) ablation studies establishing per-component necessity, (3) hyperparameter sensitivity analysis, (4) effect-size reporting with multiple-comparison correction, (5) computational complexity analysis with empirical wall-clock validation, (6) calibration-curve analysis, and (7) real-world proxy validation on a non-synthetic dataset.

Headline results: VERDICT WEIGHT outperforms Dempster-Shafer, Naive Bayes, Simple Averaging, and Max Voting on all four headline metrics (reliability error, AUC, Brier score, ECE) on a 120-record real-world proxy dataset constructed from NIST NVD and CISA KEV. The reliability-error improvement over Simple Averaging is approximately **9.6&times;** (REL = 0.0019 vs. 0.018). All differences are statistically significant under Bonferroni correction with effect sizes (Cohen's d) in the meaningful range. Ablation studies establish that each of the eight streams contributes a measurable and statistically significant share of the total improvement.

The paper documents one important dataset artifact (AUC values approaching 1.0 on this dataset reflect label-evidence coupling rather than a generalization claim) and enumerates the limitations of the empirical work in detail. The framework's case rests on REL, not AUC.

## Reproducing the results

Every numerical claim in the paper is reproducible from the published artifact:

```bash
git clone https://github.com/Odingard/verdict-weight.git
cd verdict-weight
pip install -e ".[benchmarks]"

# Head-to-head
python -m verdict_weight.benchmarks.head_to_head

# Ablations
python -m verdict_weight.benchmarks.ablation

# Real-world proxy (snapshot mode)
python -m verdict_weight.benchmarks.real_world --snapshot-date YYYY-MM-DD
```

Results match the published numbers within floating-point tolerance.
