# Evidence Streams

VERDICT WEIGHT evaluates every intelligence signal across four independent evidence dimensions. Each stream captures a distinct aspect of signal reliability. Together, they form the input vector for confidence synthesis.

## Stream 1: Source Reliability (SR)

**SR in [0,1]** — The historical trustworthiness of the originating source.

Source Reliability is not a static label. It is a computed score derived from:

- The source's track record of accuracy in the relevant domain
- Institutional credibility (primary vs. secondary vs. aggregated source)
- Known compromise history or manipulation incidents
- Verification status and authentication confidence

```python
# Example SR computation
sr_score = compute_source_reliability(
    track_record_accuracy=0.87,
    institutional_tier="primary",
    compromise_history=False,
    authentication_verified=True
)
# -> 0.89
```

**Key property:** SR alone is insufficient. A high-credibility spoofed source scores SR=0.90. Without corroboration, the Doubt Index will surface this discrepancy.

---

## Stream 2: Corroboration Count (CC)

**CC in [0,1]** — Independent agreement across multiple unrelated sources.

Corroboration Count is the most adversarially robust stream. It measures:

- The number of independent sources making the same claim
- The independence of those sources (weighted to penalize shared lineage)
- The diversity of source types (institutional, OSINT, human intelligence, sensor)

```python
# CC scoring formula
from math import exp
n_independent_sources = 4
lambda_cc = 0.5
cc_score = 1 - exp(-lambda_cc * n_independent_sources)
# -> 0.865
```

**The adversarial asymmetry:** An adversary can compromise one source. Compromising four independent sources from different institutional lineages simultaneously is operationally infeasible at speed. This is why CC is the dominant predictive stream.

<Warning>
Corroboration flooding — an adversary manufacturing multiple low-quality sources — is addressed by the independence weighting. Sources sharing institutional lineage, geographic origin, or publication timing are penalized in the CC calculation.
</Warning>

## Stream 3: Temporal Decay (TD)

**TD in [0,1]** — Recency and freshness of the intelligence relative to operational context.

Intelligence degrades. The rate of degradation is domain-dependent. VERDICT WEIGHT applies context-adaptive exponential decay:

```python
import math
td_score = math.exp(-lambda_td * age_days)

# Domain-specific decay rates:
# Cybersecurity APT:     lambda = 0.15  (fast decay)
# Zero-day:              lambda = 0.30  (very fast)
# Financial fraud:       lambda = 0.20  (moderate)
# Healthcare diagnosis:  lambda = 0.02  (slow)
# Defense intelligence:  lambda = 0.10  (moderate decay)
# Autonomous vehicle:    lambda = 2.0   (near-real-time)
```

**Key property:** Stale intelligence can be highly credible and well-corroborated while being operationally irrelevant. TD prevents confident but outdated signals from driving current decisions.

---

## Stream 4: Historical Accuracy (HA)

**HA in [0,1]** — The source's track record of prior predictions in this specific domain.

Historical Accuracy is distinct from Source Reliability. A source may be institutionally credible (high SR) but have a poor track record in a specific domain (low HA). Both are measured independently.

```python
# HA scoring formula
correct_predictions = 87
total_predictions = 100
ha_score = correct_predictions / total_predictions
# -> 0.87
```

With Laplace smoothing for new sources with limited history:
```python
# Laplace-smoothed HA
ha_score = (correct_predictions + 1) / (total_predictions + 2)
# Prevents HA=0 or HA=1 on sparse data
```

## Stream Interaction and the Doubt Index

The four streams are designed to be **independent**. When they agree, confidence is high. When they diverge, the Doubt Index rises — flagging the signal for review before it reaches the decision layer.

| Pattern | SS | DI | Interpretation |
|---|---|---|---|
| All streams high | High | Low | Act immediately |
| All streams low | Low | Low | Discard / monitor |
| High SR, low CC | Medium | **High** | Potential adversarial injection |
| High SR/CC, low TD | Medium | Medium | Credible but stale |
| Mixed across all | Low-Med | **High** | Contradicting signals — investigate |

This interaction is what makes VERDICT WEIGHT adversarially robust. It is not enough to have a credible source. The signal must survive cross-stream validation.
